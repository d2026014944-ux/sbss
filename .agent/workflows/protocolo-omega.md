## 🏛️ THE DOCUMENTATION TRINITY (MANDATORY)

Before writing any code, ensure these three artifacts are synchronized:

1. **CLAUDE.md** (The Absolute Truth): Stack, architecture, variables, infra, and the core problem domain.
2. **tasks/todo.md** (The Execution Engine): Verifiable tasks, progress tracking, and sub-task breakdown.
3. **tasks/lessons.md** (The Evolution Loop): Dissected errors and permanent rules to prevent repetition.

---

## 🧠 FUNDAMENTAL PRINCIPLES (FEYNMAN & ANTI-VIBE)

1. **Feynman Filter**: If you cannot explain the architecture or bug in simple, lay terms, you don't understand it.
2. **Root Cause Discovery**: No "band-aids". Isolate, dissect, and explain the mathematical cause root before fixing.
3. **Anti-Vibe Coding**: ZERO tolerance for "seems to work". All code must be validated and proven.
4. **Elegant Simplicity**: Implement minimum functional code. Refactor for elegance once the domain truth is known.

---

## ⚙️ WORKFLOW PHASES

### PHASE 1: Domain Mastery & Planning (Plan Node)
1. **Problem Deconstruction**: AI must explain the exact problem to the human in simple terms (Feynman Style).
2. **Plan standard node**: For any task > 3 steps, update `tasks/todo.md`.
3. **Immediate Re-plan**: Stop the moment paths become ambiguous or wrong.

### PHASE 2: Rigid TDD Development
1. **Absolute TDD**: Write tests BEFORE features. Tests must express business rules simply.
2. **Active Rejection**: AI will NOT write code without passing tests or a clear test plan first.
3. **Human Control**: If AI loops or complicates, Human takes over, corrects, and restarts.

### PHASE 3: Refactoring & Verification
1. **Senior Audit**: Would a senior engineer approve this simplicity?
2. **Cross-Testing**: Validate communication between modules/services.
3. **Real Proof**: Run scripts, verify logs, and provide evidence of success.

### PHASE 4: Feynman Debugging & Evolution
1. **Declare Root Cause**: AI must state the cause root in 1-2 simple sentences before fixing.
2. **Autonomous Cure**: Once diagnosed, inject the exact, elegant fix.
3. **Loop Closure**: Update `tasks/lessons.md` with the new wisdom learned.
